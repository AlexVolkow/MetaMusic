ID3-Tag Specification V1.1 by Michael Mutschler

Overview

Since I always missed a correct specification of the ID3-Tag for the MP3s, here is mine. I don't know where the original one came from, and there is noone who officially supports it, I want to make it with this page.

The ID3-Tag is an information field for MPEG Layer 3 audio files. Since a standalone MP3 doesn't provide a method of storing other information than MP3 related, the ID3 allows to do this. This is no official standard, but I hope this file helps all you guys out there.

I have made an addition to the original ID3-Tag to store the information of the track. The below for this feature. This is the reason why I set the version to 1.1!

Implementation

The Information is stored in the last 128 bytes of an MP3. The Tag has got the following fields, and the offsets given here, are from 0-127

FieldLengthoffsets
Tag30-2
Songname303-32
Artist3033-62
Album3063-92
Year493-96
Comment3097-126
Genre1127

The string-fields contain ASCII-data, and the codepage used for it is ISO-Latin 1 Strings which are smaller than the 30 (4) characters are padded with zero-bytes.

  • Tag: the tag is valid if this field contains the string "TAG". This has to be uppercase!
  • Songname: this field contains the title of the MP3. (string as above)
  • Artist: this field contains the artist of the MP3. (string as above)
  • Album: this field contains the album where the MP3 comes from. (string as above)
  • Year: this field contains the year when this song has originally been released. (string as above)
  • Comment: this field contains a comment for the MP3. (string as above) I have made an addition to this field, to store the track number of this songs. See Track addition below.
  • Genre: this byte contains the offset of a genre in a predefined list the byte is treated as an unsigned byte. The offset is starting from 0. There are currently defined 115 genres which I am not responsible for! Because there is no default-Field, I use an unused value (255) to indicate this. The current list is shown below.

Track addition

I always missed the feature of specifying the track of the module from the album it originally comes from. Or how to obtain the original order of all MP3s of an Album.
Now I found a solution, which is compatible to the ID3-Tag, and let�s you add a track-number: This is done due to the cost of the comment field beeing reduced to 28 characters, or leaving the full 30 characters to the comment, and having no track-info. I use the comment-field, because a) it is rarely used, b) comment is too short to be useful, so it doesn�t matter if it�s 30 or 28 chars long.

For all of you programmers, this is the method of doing it:
You simply need to check if character 28 in the comment-field is zero, then character 29 contains the track-number. Treat this byte as an unsigned integer. Here is some pseudo code to do this: 

	Comment: Array [0..29] of unsigned byte;
	Track: unsigned byte;

	if (Comment[28] = 0) then
		Track := Comment[29];
	else Track := 0;

This will get a track of zero even if the track cannot be read. And a track of zero should be treated as the unknown Track. It even works correct, if the comment is 30 chars long!
